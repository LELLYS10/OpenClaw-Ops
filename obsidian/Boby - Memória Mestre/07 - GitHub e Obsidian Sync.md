# GitHub e Obsidian Sync

## Ideia central
Usar o Obsidian como central humana de organização e o GitHub como base versionada para o BOBY/OpenClaw consumir.

## Regra prática
- Obsidian organiza
- GitHub consolida
- OpenClaw consulta

## Estrutura recomendada
- Vault local: `JARVIS-SYSTEM-OBSIDIAN`
- Repo privado: `knowledge-hub`

## O que sobe para o GitHub
- notas úteis
- checklists
- estrutura de agentes
- memória consolidada
- contexto dos sistemas

## O que não deve subir
- segredos
- tokens
- chaves
- arquivos sensíveis sem revisão

## Nome fácil para lembrar
- local: `Boby - Memória Mestre`
- remoto: `knowledge-hub`
