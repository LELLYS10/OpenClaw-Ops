# Estrutura de Memória

## Camadas
### 1. Memória diária
- `daily/YYYY-MM-DD.md`
- fatos do dia
- pedidos
- ações
- pendências

### 2. Memória longa
- `MEMORY.md`
- só fatos duráveis
- preferências
- regras
- contexto persistente

### 3. Projetos
- `projects/`
- contexto por projeto
- backlog
- status

### 4. Pessoas
- `people/`
- perfis importantes
- relações
- contexto útil

### 5. Checklists
- `checklists/`
- boot
- revisão
- heartbeat
- fechamento

## Regra central
Memória não pode ficar só no modelo.
Se for importante, tem que virar arquivo.

## Fluxo recomendado
1. fato aparece
2. vai para memória diária
3. se for durável, sobe para memória longa
4. se for de projeto, também vai para projeto correspondente
